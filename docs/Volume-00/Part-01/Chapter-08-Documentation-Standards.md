# Chapter 8 — Documentation Standards

## Purpose
Documentation is a long-term asset. Every document in ENDFRAME OS should be understandable months or years after it is written.

## Core Principles
- Write for your future self.
- Prefer clarity over cleverness.
- Keep one topic per document.
- Update documentation alongside code or projects.
- Link related concepts.

## Recommended Structure
1. Purpose
2. Background
3. Main Content
4. Examples
5. References
6. Next Steps

## Writing Guidelines
- Use descriptive headings.
- Keep paragraphs concise.
- Prefer bullet lists for procedures.
- Include diagrams where relationships matter.
- Record assumptions and limitations.

## Naming Conventions
- Use kebab-case for file names where appropriate.
- Prefix chapters with ordered numbers.
- Use consistent terminology throughout the repository.

## Documentation Checklist
- Accurate
- Current
- Searchable
- Linked
- Actionable
- Versioned

Well-maintained documentation compounds in value over time and reduces future maintenance effort.
